# Sprint Lifecycle

GStack's core operating model is:

```text
Think -> Plan -> Build -> Review -> Test -> Ship -> Reflect
```

## Think

Clarify the real problem, not just the requested feature.

Example workflows:

- Office Hours
- CEO Review

## Plan

Turn the intent into architecture, scope, tests, and design direction.

Example workflows:

- Autoplan
- Engineering Review
- Design Review
- Spec

## Build

Implement the selected plan in small, reviewable steps.

Example workflows:

- Design HTML
- Codex second opinion
- Guard / Freeze for safety

## Review

Catch production-grade bugs, unnecessary complexity, and design issues.

Example workflows:

- Review
- Design Review
- DevEx Review
- CSO Security Review

## Test

Use real browser and realistic flows to validate the product.

Example workflows:

- QA
- QA Only
- Browse
- Benchmark

## Ship

Prepare release, PR, merge, deploy, and production verification.

Example workflows:

- Ship
- Land and Deploy
- Canary

## Reflect

Record learnings so the next sprint gets better.

Example workflows:

- Retro
- Learn
- Context Save
