# Specialists

Workflow Wiki organizes workflows by the specialist mindset they represent.

## Initial Specialist Map

| Specialist | Main Stage | Example Workflows |
|---|---|---|
| CEO / Founder | Think | Office Hours, CEO Review |
| Engineering Manager | Plan | Engineering Review, Autoplan |
| Designer | Plan / Review | Design Consultation, Design Shotgun, Design Review |
| Staff Engineer | Review | Review, Investigate |
| QA Lead | Test | QA, QA Only, Browse |
| Security Officer | Review | CSO |
| Release Engineer | Ship | Ship, Land and Deploy, Canary |
| Technical Writer | Reflect | Document Release, Document Generate |

Each specialist page should eventually explain:

- when to use the specialist
- inputs required
- decisions they own
- workflows they run
- outputs they produce
- handoff to the next specialist
